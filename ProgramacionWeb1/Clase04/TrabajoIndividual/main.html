Tema 01

HTML

HTML es el lenguaje base para estructurar la información en la web. En PERBIONIC lo usamos para construir interfaces de consulta clínica, catálogos de prótesis y formularios de evaluación técnica.

Ejemplo: Lorem ipsum dolor sit amet, consectetur adipiscing elit. Vestibulum vitae arcu venenatis, sollicitudin turpis a, facilisis justo.
Tema 02

HTML Introduction

Una página HTML está compuesta por elementos que representan encabezados, párrafos, tablas e imágenes. Esta introducción permite entender cómo presentar datos biomédicos de forma clara y accesible.

Tema 03

HTML Editors

Se puede trabajar con editores simples y avanzados. Para el equipo de PERBIONIC, editores con autocompletado agilizan la documentación de protocolos y la creación de plantillas clínicas.

Tema 04

HTML Basic

La estructura básica incluye <!DOCTYPE html>, <html>, <head> y <body>. Con esta base se pueden montar paneles informativos para pacientes y profesionales.

Tema 05

HTML Attributes

Los atributos añaden información a los elementos (por ejemplo, href, src, alt). Son esenciales para accesibilidad y trazabilidad de contenido técnico.

Tema 06

HTML Heading

Los encabezados de h1 a h6 ordenan la jerarquía del contenido. Un reporte biomédico bien estructurado ayuda a que el personal clínico localice datos críticos rápidamente.

Tema 07

HTML Paragraphs

Los párrafos separan ideas y explicaciones. Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer dapibus massa vel congue tincidunt.

Tema 08

HTML Styles

Con estilos inline o clases CSS, se puede destacar información sensible, como alertas de mantenimiento preventivo en equipos médicos.

Nota: Mantener consistencia visual mejora la usabilidad.

Tema 09

HTML Formatting

Etiquetas como <b>, <strong>, <i> y <mark> ayudan a enfatizar términos clínicos, advertencias y resultados de pruebas.

Tema 10

HTML Quotations

“La tecnología médica debe ser funcional, inclusiva y centrada en el paciente.”

Se pueden usar etiquetas de cita para resaltar principios institucionales o referencias de investigación.

Tema 11

HTML Comments

Los comentarios permiten documentar el código sin mostrar contenido al usuario final. Son útiles cuando varios desarrolladores colaboran en módulos clínicos.

Tema 12

HTML Colors

Los colores pueden definirse por nombre, HEX, RGB o HSL. Para PERBIONIC se priorizan tonos azules por su asociación con confianza, salud y tecnología.

Tema 13

HTML CSS

CSS permite separar presentación y contenido. Con ello, un mismo sistema puede adoptar temas para laboratorio, consulta externa o administración.

Tema 14

HTML Links

Los enlaces conectan recursos internos y externos, como manuales de uso o campañas de prevención.

Visitar referencia base de W3Schools

Tema 15

HTML Favicon

El favicon ayuda a identificar rápidamente una pestaña del sistema, especialmente cuando el personal clínico trabaja con múltiples aplicaciones abiertas.

Tema 16

HTML Page Title

El título de la página mejora la organización del navegador y también aporta contexto en buscadores y marcadores.

Tema 19

HTML Block e Inline

Los elementos de bloque ocupan todo el ancho disponible, mientras que los inline se adaptan al contenido. Esta diferencia ayuda al diseñar fichas técnicas legibles.

Tema 20

HTML Div

El contenedor div se utiliza para agrupar componentes y aplicar estilos. Es útil para organizar módulos de pacientes, dispositivos y reportes.

Tema 21

HTML Classes

Las clases permiten reutilizar estilos en múltiples componentes. Por ejemplo, tarjetas de evaluación con un mismo formato visual.

Tema 22

HTML Id

El atributo id identifica un elemento único, ideal para crear anclas de navegación o disparar comportamientos con JavaScript.

Tema 23

HTML Buttons

Los botones permiten acciones como guardar evaluaciones o enviar formularios de contacto.

Tema 24

HTML Iframes

Un iframe permite incrustar contenido externo, como videos explicativos de rehabilitación.

Tema 25

HTML JavaScript

JavaScript agrega interactividad al HTML: validación de formularios, alertas, cálculos simples y actualización dinámica de métricas clínicas.

Tema 26

HTML File Paths

Las rutas relativas y absolutas ayudan a cargar imágenes, íconos y scripts correctamente. Una ruta bien definida evita errores en despliegues internos.

Tema 27

HTML Head

Dentro de head se declaran metadatos, estilos, favicon y título. Es la base para preparar una página optimizada y bien identificada.

Tema 28

HTML Layout

El layout define cómo se distribuyen las secciones visibles. En esta página se utiliza un esquema con navegación lateral y contenido principal para facilitar el estudio secuencial de HTML.
